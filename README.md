# 🩺  BioMed Healthcare Equipment System Analysis Modelling
## Report Overview
This report provides a comprehensive analysis of the current system supporting the Live Glucose Monitoring (LGM) device, outlining stakeholder needs, functional and non-functional requirements, and adaptive alert settings. 

A central focus of the report is the use of system diagrams, which play a critical role in illustrating workflows, data interactions, and relationships between components. These visual models not only clarify complex processes but also ensure that requirements are effectively communicated to both technical and non-technical stakeholders.

## The report includes:
Part A
- Stakeholders Analysis
- Patients inclusion in the system requirements & Interaction between Medical Professional and LGM Device
- Requirements determination
- Functional & Non-Functional Requirements
- Adaptive Alert Settings
- Recommendations

Part B
- **Context Diagram** – to show the overall system environment and external interactions.
- **Event Table** – to capture events that trigger system responses.
- **Data Flow Diagram Fragments** (View History, Send Messages) – to represent specific functional processes.
- **Entity Relationship Diagram (ERD)** – to define data entities and their relationships.
- **Level 1 Data Flow Diagram (DFD)** – to detail the flow of information within the system.
